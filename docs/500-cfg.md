# Configuration

batipanel stores its configuration in `~/.batipanel/`. Here's what lives there and how to tweak it.

## Directory Structure

```
~/.batipanel/
  bin/            # CLI scripts
  lib/            # core modules
  layouts/        # layout templates
  config/         # tmux.conf and theme configs
  projects/       # registered project scripts
  completions/    # shell tab completion
  config.sh       # user settings (key=value)
  VERSION         # installed version
```

## config.sh

The main settings file at `~/.batipanel/config.sh`. It's a simple key=value format (never sourced directly for security).

| Key | Default | Description |
|-----|---------|-------------|
| `DEFAULT_LAYOUT` | `7panel` | Layout used when none is specified |
| `BATIPANEL_THEME` | `default` | Color theme name |
| `BATIPANEL_ICONS` | `0` | Enable Nerd Font icons in eza (`0` or `1`) |
| `BATIPANEL_ITERM_CC` | unset | iTerm2 native integration (`0` or `1`) |

Edit via CLI (recommended):

```bash
b config layout 6panel
b theme dracula
b config iterm-cc on
```

Or edit the file directly:

```bash
vim ~/.batipanel/config.sh
```

## tmux.conf

batipanel installs its own tmux config at `~/.batipanel/config/tmux.conf` and sources it from `~/.tmux.conf`. The config sets up:

- Keyboard shortcuts (Alt+h/j/k/l navigation, Alt+f zoom, etc.)
- Mouse support
- Vi-style copy mode
- Pane border styling
- Status bar with Powerline arrows
- Theme colors

If you have an existing `~/.tmux.conf`, batipanel appends a `source-file` line. Your custom settings are preserved.

To reload tmux config manually:

```bash
# Inside tmux
Ctrl+b then r

# From shell
tmux source-file ~/.tmux.conf
```

## Environment Variables

| Variable | Description |
|----------|-------------|
| `BATIPANEL_HOME` | Install directory (default: `~/.batipanel`) |
| `BATIPANEL_DEBUG` | Set to `1` for debug logging |
| `BATIPANEL_NO_COLOR` | Set to `1` to disable colors |
| `NO_COLOR` | Standard no-color flag (also disables colors) |

Use `--debug` for one-off debugging:

```bash
b myproject --debug
```

## Project Scripts

Each registered project gets a script at `~/.batipanel/projects/<name>.sh`. These are auto-generated by `b new` and contain the project path and layout loader. You generally don't need to edit these.

```bash
b new myproject ~/code/myproject    # creates ~/.batipanel/projects/myproject.sh
b ls                                # list all registered projects
```

## Tab Completion

The installer sets up tab completion for bash and zsh. If it's missing:

```bash
# Bash
source ~/.batipanel/completions/batipanel.bash

# Zsh
source ~/.batipanel/completions/batipanel.zsh
```

Verify with `b doctor` — it checks for tab completion.

## Resetting Config

```bash
b reset
```

This removes config.sh, registered projects, and kills tmux sessions. The setup wizard runs again on next launch.
