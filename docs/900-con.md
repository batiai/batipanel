# Contributing

Want to help improve batipanel? Here's how to get started.

## Dev Setup

```bash
git clone https://github.com/batiai/batipanel.git
cd batipanel
bash install.sh
```

This installs batipanel from your local clone. Changes to files in the repo take effect immediately (the installer copies files to `~/.batipanel/`).

## Making Changes

1. Fork the repo and create a branch:
   ```bash
   git checkout -b my-feature
   ```

2. Make your changes

3. Run the required checks:
   ```bash
   # Syntax check (must pass)
   bash -n lib/*.sh bin/*.sh install.sh uninstall.sh layouts/*.sh

   # ShellCheck (must pass)
   shellcheck -s bash lib/*.sh bin/*.sh install.sh uninstall.sh layouts/*.sh
   ```

4. Test the install/uninstall cycle:
   ```bash
   bash install.sh
   b doctor           # should show all green
   bash uninstall.sh
   ```

5. Commit and open a PR

## Code Style

- **2-space indentation** in all shell scripts
- All scripts start with `set -euo pipefail`
- Error messages use color variables (`$RED`, `$GREEN`, `$YELLOW`, `$BLUE`, `$NC`)
- Comments and user-facing messages in **English**
- Follow existing patterns тАФ check `lib/core.sh` for examples
- Keep files under **300 lines** when possible

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new layout for ultrawide monitors
fix: handle spaces in project paths
refactor: extract theme loading into separate module
docs: update layout diagrams
chore: bump version to 0.5.0
```

## Adding a Layout

1. Copy an existing layout:
   ```bash
   cp layouts/7panel.sh layouts/myname.sh
   ```

2. Edit the panel splits and tool assignments. Use the helper functions:
   - `init_layout` тАФ create the tmux session
   - `_split` тАФ split panes (compatible wrapper)
   - `run_claude`, `run_lazygit`, `run_monitor`, `run_filetree`, `run_remote` тАФ launch tools
   - `label_pane` тАФ set pane title

3. Add your layout to the copy list in `install.sh`

4. Test on a small terminal (4panel baseline) and a large one

See [Layouts](300-lay.md) for diagrams of existing layouts.

## Adding a Theme

Theme colors are defined in `lib/themes-data.sh`. Each theme needs:

- `_get_theme_colors` тАФ tmux status bar colors (15 values)
- `_get_theme_desc` тАФ short description
- `_get_theme_terminal_colors` тАФ terminal OSC colors (7 values)
- Add the name to `BATIPANEL_THEMES`

## Project Structure

```
bin/
  cli.sh         # npm/npx entry point
  start.sh       # main CLI (command routing)

lib/
  common.sh      # module loader
  core.sh        # colors, utilities, config parsing
  validate.sh    # tmux version, session name, terminal size checks
  layout.sh      # layout framework, tool launchers
  session.sh     # tmux session lifecycle
  project.sh     # project registration, config commands
  doctor.sh      # system diagnostics
  wizard.sh      # first-run setup guide
  themes-data.sh # theme color definitions

layouts/         # layout templates (4panel through 8panel)
```

Module loading order: `common.sh` -> `core.sh` -> `validate.sh` -> `layout.sh` -> `session.sh` -> `project.sh` -> `doctor.sh` -> `wizard.sh`

## Reporting Issues

Open an issue at [github.com/batiai/batipanel/issues](https://github.com/batiai/batipanel/issues) with:

- Your OS and terminal emulator
- `b doctor` output
- Steps to reproduce

## Security

- Never use `eval` on user input
- Never `source` untrusted files
- Always validate and escape user-provided paths
- Config parsing uses a safe key-value reader (see `lib/core.sh`)

See the [Configuration](500-cfg.md) docs for how config.sh is parsed safely.
